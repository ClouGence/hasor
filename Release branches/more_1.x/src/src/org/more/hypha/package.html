beans v1.2系统结构图
历史版本：
hypha v1.0 [beans v2.0](版本alpha4)：
  1.更名组建并升级到2.0。
  2.全新的xml解析支持可以使菌丝的配置文件合并到任何其它系统的xml配置文件中。
  3.精确的模型定义描述和扩展支持。
  4.插件化Aop支持。
  5.全面支持新版classcode底层功能。
  6.支持各个层面的事件机制。
  7.可扩展的xml配置文件系统和解析器。
  8.加入了EL的支持
  	EL注入、EL调用、扩展内置EL对象、、、
  9.取消了v1.0版本中的关键字注入，并且使用EL形式增强了该输入器。
  	params		使用在getBean时候传递的参数对象进行注入
  	attribute	表示绑定在上下文中的属性信息。
  	context		ApplicationContext类型对象。
  10.取消了Ioc、Fact、Export三种注入方式，合并为同一种EL注入(属性注入)。
  11.支持bean的方法声明。
  12.加入了作用域的支持。
  13.加入了事件机制。
  14.加入了脚本机制。
  15.加入了扩展点，通过扩展点可以参与hypha内部的运转。
  	
beans v1.2(版本alpha3)：
  1.增加注解配置支持，至此xml配置与注解支持可以同时进行。xml的优先级高于注解。
  2.可以注解配置Bean。
  3.可以注解配置Bean的构造方法参数注入
  4.可以注解配置Bean的Aop链。
  5.可以注解配置附加接口实现，但是所有附加接口都使用一个接口处理委托。如果要一个接口对应一个委托只有xml配置支持。
  6.可以注解配置属性注入。
  7.注解配置支持以定义的基本类型和四种引用类型。
  8.注解配置需要通过xml配置anno:anno标签。
  9.通过anno标签可以开启注解配置以及配置beans的扫描路径。
  10.重写了Resource接口的实现类。

beans v1.1(版本alpha2)：
  1.增加，beans在初始化时候会读取xsd验证xml合法性。
  2.调整一些Bug。

beans v1.0(版本alpha)：
  1.独立的Ioc、Aop框架，它的配置文件结构参考了Spring的配置文件结构。
  2.形成了三大核心组建1)factory负责创建、2)injection负责注入、3)propparser负责属性解析。
  3.支持构造方法注入和属性注入。
  4.提供了三种注入形式【Ioc、Fact、Export】
	Ioc：如果需要注入大量依赖bean则使用Ioc性能最好。
	Fact：如果注入大量基本数据类型和少量的引用类型，并且这些引用类型多数配置了单态则Fact可以发挥极致性能，如果注入的数据全部为基本数据类型Fact的速度接近传统get/set方法直接调用注入。
	Export：导出式注入为了追求比Fact更快的注入可以使用该方式。Export方式注入整个注入过程由编程人员在ExportInjectionProperty接口中实现。
  5.提供了两种bean创建方式【Factort、New】
	Factort：对工厂创建方式的支持。
	New：普通构造方法创建支持。
  6.aop提供过滤器链支持，过滤器需要实现AOPInvokeFilter接口。
  7.支持关键字注入（四种引用注入）。
	refBean：使用配置的其他bean对象，PRV_Bean.name。
	{#attName}：使用BeanFactory接口的父接口IAttribute中的对象注入，PRV_ContextAtt{#name}。
	{@number}：使用在getBean时候传递的参数对象进行注入，PRV_Param{@number}。
	{$mime}：使用附加在bean定义中的属性注入，PRV_Mime{$name}。
	提示1：{#this}会将BeanFactory实现对象注入到bean中，目前版本不支持关键字行为自定义。
	提示2：关键字行为是指被“{}”环绕的第一个字符，more可以通过第一个字符来确定注入数据来源。
	提示3：{$name}与{#name}不同{$name}依照最近的层级结构一次判断是否有这个属性如果没有向上一层寻找。
  8.可以通过addImpl标签来向bean附加一个接口实现，而这个bean可以事先没有实现这个接口。注意：已经实现的接口addImpl将失效，不同接口的同名方法将只有一个被保留。
  9.支持注入Map，Set，List，Array数据类型。
  10.支持的基本类型包括: int、byte、char、double、float、long、short、boolean、String。
  11.可以通过xsd验证文档结构，more不会验证xml合法性。
  12.支持bean定义静态缓存和动态缓存。利用这两个缓存可以增加运行效率。
